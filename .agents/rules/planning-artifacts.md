# Planning Artifacts

When you start or finish a task, you MUST copy the content of the `task.md`, `implementation_plan.md`, and `walkthrough.md` files (or provide a combined summary like an `initial_route_handling.md` file) into a folder named `planning/<feature_name>` in the root of the project directory.

For example, if you are working on the "auth" feature, you should create `planning/auth/<task_name>.md`. You must ALWAYS keep this up-to-date and maintain these files in the `planning` directory for every task you accomplish so there is a persistent record of the process.

This is a strict requirement for all tasks processed.
